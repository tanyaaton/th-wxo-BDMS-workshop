Welcome to the Lab Guide

This guide walks you through four hands-on labs covering IBM watsonx Orchestrate: setting up your environment, deploying a pre-built Email agent, building a RAG agent with a knowledge base, and composing a multi-agent system. Follow each section in order.

Before You Begin

Setup

Complete this once before starting any lab. You will register for a free trial, access the IBM dashboard, and launch a watsonx Orchestrate instance.

Step 1 — Register for a Free Trial

  1. 1
    Go to the watsonx Orchestrate product page

    Open your browser and navigate to the IBM watsonx Orchestrate product page.

    Open IBM watsonx Orchestrate
  2. 2
    Start the free trial

    Scroll to the bottom of the page, click Free Trial, and fill in your account information to register.

Step 2 — Launch Your Instance

  1. 3
    Go to the IBM dashboard

    After registration, navigate to your IBM product dashboard.

    Open IBM Dashboard
  2. 4
    Find watsonx Orchestrate and click Launch

    Under My Products, locate IBM watsonx Orchestrate and click Launch.

  3. 5
    Create a new instance

    Click Create new instance and select the us-east region.

  4. 6
    Launch the instance

    Click Launch. You should now see the watsonx Orchestrate front page and are ready to start the labs.

    You're all set

    Once you can see the watsonx Orchestrate home screen, proceed to Lab 1.

Lab 1

Pre-built Email Agent

Deploy and configure a pre-built Google Email Manager agent from the watsonx Orchestrate Discovery catalogue, connect it to Gmail via OAuth2, and send your first AI-generated email.

Step 1 — Find the Email Manager Agent

  1. 1
    Open the Discovery page

    In watsonx Orchestrate, navigate to the Discovery page using the left-hand navigation.

  2. 2
    Search for the Google pre-built agent

    In the search bar, type google to filter the catalogue. Locate and select Email Manager.

Step 2 — Configure the Gmail Connection

  1. 3
    Open the Tools > Connection settings

    Inside the Email Manager agent, click Tools in the top tabs, then select Connection.

  2. 4
    Enter the Gmail OAuth2 credentials

    Fill in the connection form using the values below. You will need a Client ID and Client Secret from your Google Cloud OAuth2 App (step 4 of the Google setup).

    Authentication type OAuth2 Authorization Code
    Server URL https://gmail.googleapis.com/
    Token URL https://oauth2.googleapis.com/token
    Authorization URL https://accounts.google.com/o/oauth2/v2/auth
    Client ID (from your Google Cloud OAuth2 App)
    Client Secret (from your Google Cloud OAuth2 App)
    Scope https://mail.google.com/
    Token request field prompt = consent
    Auth request fields base_url = https://gmail.googleapis.com/
    access_type = offline
    Where do I get the Client ID and Secret?

    Go to the Google Cloud Console, create or select a project, navigate to APIs & Services > Credentials, and create an OAuth 2.0 Client ID. Make sure the Gmail API is enabled for your project.

Step 3 — Test the Email Agent

Once the connection is configured, test the agent by sending yourself an email using the prompt below.

Sample prompt — send a test email
send email to <your email> with topic: Your dental appointment content: we have already scheduled your appointment
Tip

Replace <your email> with your actual email address. The agent will compose and send the message through your connected Gmail account.

Lab 2

RAG Agent with Knowledge Base

Build an agent from scratch, attach a PDF knowledge base, and enable retrieval-augmented generation (RAG) so the agent can answer questions grounded in your documents. You can complete this lab using either the UI or the ADK CLI.

Option A — UI Approach

  1. 1
    Create a new agent from scratch

    In watsonx Orchestrate, click Create agent and choose the from scratch option. Give the agent a name and description.

  2. 2
    Add a Knowledge Base

    Inside the agent editor, click Add Knowledge Base and choose Import file.

  3. 3
    Upload the dental department knowledge base

    Upload one of the following PDF files from the workshop repository:

    English version

    LAB_2_RAG _AGENT/policy/dental_department_knowledge_base.pdf

    Thai version

    LAB_2_RAG _AGENT/policy/dental_department_knowledge_base_th.pdf

Option B — ADK (CLI) Approach

  1. 1
    Activate your environment

    Open a terminal and run the following command, replacing the placeholders with your environment name and API key:

    Terminal — activate environment
    orchestrate env activate <your-env-name> -a <your-api-key>
  2. 2
    Run the import script

    Execute the provided import script to deploy all agent resources in one go:

    Terminal — import all resources
    bash import_all.sh
Lab 3

Multi-Agent System

Compose a multi-agent system by connecting the Email Agent from Lab 1 and the RAG Agent from Lab 2 as collaborators, then deploy and test the orchestrator agent end-to-end.

Step 1 — Build the Multi-Agent Orchestrator

  1. 1
    Create a new agent from scratch

    In watsonx Orchestrate, click Create agent and choose the from scratch option. This will be your orchestrator agent.

  2. 2
    Add collaborator agents

    In the agent editor, select the Collaborators section and add both:

    • The Email Manager Agent created in Lab 1
    • The RAG Agent (Dental Knowledge Base) created in Lab 2
  3. 3
    Deploy the orchestrator agent

    Click Deploy to publish the multi-agent system and make it available for testing.
